🎯 Étapes de Déploiement

1
Configuration Discord Bot
5 min
👤
TOI tu fais :
  1. Va sur Discord Developer Portal
  2. Clique "New Application" → nom: "Scooty"
  3. Onglet "Bot" → "Add Bot" → Copie le TOKEN
  4. Onglet "OAuth2" → "URL Generator":
    • 1. D'ABORD Scopes: ✅ bot ✅ applications.commands
    • 2. ENSUITE Bot Permissions (apparaît après): ✅ Send Messages ✅ Embed Links ✅ Read Message History
  5. Copie l'URL et invite le bot sur ton serveur
  6. Donne-moi le TOKEN Discord
🤖
MOI je peux :
  • ✅ Créer le fichier .env automatiquement
  • ✅ Configurer les variables Discord
  • ✅ Vérifier la configuration
  • ✅ Tester la connexion bot
# Je peux exécuter : cp env.example .env # Puis éditer .env avec tes tokens
2
Configuration Supabase
3 min
👤
TOI tu fais :
  1. Va sur Supabase.com
  2. Clique "New Project"
  3. Nom: "scooty-db", mot de passe fort
  4. Une fois créé, va dans "SQL Editor"
  5. Va dans "Settings" → "API" → Copie:
    • Project URL
    • anon public key
  6. Donne-moi l'URL et la clé Supabase
🤖
MOI je peux :
  • ✅ Lire le script SQL complet
  • ✅ Te donner le contenu exact à copier/coller
  • ✅ Configurer les variables Supabase dans .env
  • ✅ Vérifier la connexion à la base
# Le script SQL est dans : supabase/schema.sql # Je peux te l'afficher entièrement
3
Installation Fly.io CLI
2 min
👤
TOI tu fais :
  1. Ouvre ton terminal
  2. Copie/colle les commandes que je te donne
  3. Crée un compte Fly.io (gratuit)
  4. Confirme que ça marche
🤖
MOI je peux :
  • ✅ Exécuter les commandes d'installation
  • ✅ Vérifier que Fly.io CLI est installé
  • ✅ Tester la connexion
  • ✅ Préparer la configuration fly.toml
# Je peux lancer : curl -L https://fly.io/install.sh | sh flyctl version
4
Variables d'Environnement
1 min
👤
TOI tu fais :
  • Me donner tes tokens Discord
  • Me donner tes clés Supabase
  • Valider le fichier .env généré
🤖
MOI je peux :
  • ✅ Créer le fichier .env complet
  • ✅ Remplir toutes les variables
  • ✅ Vérifier le format
  • ✅ Tester la configuration
# Je vais créer : DISCORD_TOKEN=ton_token CLIENT_ID=ton_client_id SUPABASE_URL=ton_url SUPABASE_ANON_KEY=ta_key # ... et toutes les autres variables
5
Déploiement Final
3 min
👤
TOI tu fais :
  • Autoriser les commandes Fly.io
  • Confirmer le déploiement
  • Tester le bot sur Discord
🤖
MOI je peux :
  • ✅ Lancer le script de déploiement
  • ✅ Exécuter npm install & build
  • ✅ Configurer Fly.io
  • ✅ Déployer en production
  • ✅ Vérifier que le bot est en ligne
# Je vais exécuter : chmod +x deploy.sh ./deploy.sh # Ou en manuel : npm install npm run build flyctl launch flyctl deploy

🚨 Points d'Attention

⚠️ Important :
  • Garde tes tokens secrets, ne les partage jamais publiquement
  • Le fichier .env ne sera jamais committé (dans .gitignore)
  • Fly.io est gratuit pour commencer, mais surveille les limites
✅ Après déploiement :
  • Ton bot sera accessible 24/7 sur internet
  • Les commandes /scout, /check, /stats, /help marcheront
  • La base de données stockera automatiquement les scouts
  • Tu pourras voir les logs et métriques sur Fly.io

🧪 Tests à Faire

  1. Sur Discord, tape /scout → vérifier l'auto-complétion

  5. Tester l'aide avec /help
